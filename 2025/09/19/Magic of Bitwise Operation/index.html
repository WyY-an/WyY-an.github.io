位运算魔法

位运算介绍

为什么说位运算是魔法呢，因为它真的很神奇。很难想到但又十分合理，原理简单但操作极多。

位

要了解什么是位运算，首先要了解什么是位。

位 (bit) 是计算机用于存储数据的最小单位，一个位只代表两种状态 0 和 1 。

8个位组成一个字节 (byte) ，例如 00000000 。

而在 C语言 中，一个int的内存大小是 4 个字节，也就是 32 位，最左边的一位代表符号(sign)，如果符号位为 0 则代表正，为 1 则代表负。也就是说：一个整形代表数值的有 31 位，因此 int 的最大值为 $2^{31}-1$ 。除了int (整形) 以外，我们还有 unsigned int (无符号整型)，即将符号位看做表示数值的位，因此unsigned int类型的最大值为 $2^{32}-1$ 。

例如 int 类型的 3 在内存中的值就是 $00000000\ 00000000\ 00000000\ 00000011$​。-2 在内存中是 $10000000\ 00000000\ 00000000\ 00000010$ 。

真值

真值，即 bool 类型的两个值：true 和 false 。代表真和假。

在我们的 if、while和for 的条件判断时，判断的就是真值。如果为真则运行，为假则停止。

特别注意：在数与真值的关系中，0代表假，其余一切数字都代表真。后面在取反和按位取反中会提及

位运算

接下来将介绍位运算的每一种运算。

1.位与

在C/C++及大部分编程语言中，位与用 & 表示。

如果是对真值判断，两边都为真结果才是真，即：true & true 的结果为 true。

注意到，在真值位与的判断中，如果第一项为 false，那么无序判断第二项，而仅仅使用 & 会导致前后都算，因此我们可以使用 && 去位与真值，用 && 运算，如果第一项为假，那么不再判断第二项。

如果是对数值判断，那么会按位进行运算，并且前后的数据类型一致，一般只能对整数进行位与运算。接下来介绍按位运算。

在前面我们知道，整数在内存中都是以位的形式存储的拿 5 和 3 举例。

5: 00000000 00000000 00000000 00000101

3: 00000000 00000000 00000000 00000011

接下来对每一位进行与运算，倒数第三位分别为 1 和 0 ，与后结果为 0，倒数第二位分别为 0 和 1，与后结果为 0，最后一位都是 1，结果为 1。

因此我们得到 5 & 3 = 1 (00000000 00000000 00000000 00000001)。

2.位或

和位与不同的是，位或为有一边为真即为真。对真值判断时，如果第一项为真，第二项则不需要判断，因此可以用 || 来简化运算。

对数值运算，也是将数字按位或就可以，仍然以 5 | 3 举例。

接下来对每一位进行与运算，倒数第三位分别为 1 和 0 ，或后结果为 1，倒数第二位分别为 0 和 1，或后结果为 1，最后一位都是 1，结果为 1。

因此我们得到 5 | 3 = 7 (00000000 00000000 00000000 00000111)。

3.异或

跟 位与 和 位或 不同的是，异或必须要判断两项，因为异或的计算原理是： 真值不相同为真，相同则为假。用符号 ^ 表示。

例如 false ^ true 的结果为 true，而 true ^ true 的结果为 false。

同样，对数值的异或也是按位异或。大家可以尝试计算 5 ^ 3 。最终结果为 6 。

4.取反

取反的符号为 ! ，顾名思义 !true 为 false ， !false 为 true。

注意对数值取反时，!0 为 true ，其他数值取反都为 false。

5.按位取反

和取反不同，按位取反是指将数字转为二进制后，对每一位进行取反，注意：得到的结果是补码。用符号 ~ 表示。

按位运算需要涉及到 原码 和 补码 两个概念。读者自行了解。这里对 ~1 进行举例。

1：(0000000 00000000 00000000 00000001)。

按位取反后得到 11111111 11111111 11111111 11111110 。

之后转化为原码得到 10000000 00000000 00000000 00000001 + 1 = 10000000 00000000 00000000 00000010

即 ~1 = -2。

左移与右移

左移为 << ，右移为 >>。

左移表示将所有位向左移动一位，右边用 0 补全。例如：1<<1 = 2。|… 01|左移变为|…10|。

右移则代表所有位向右移动一位，左边用 0 补全。不难发现 右移操作 和 int 类型的数字除二 得到的结果一致。
